# Steering スキル

ステアリングファイル(`.steering/`)に基づいた実装を支援し、tasklist.mdの進捗管理を行うスキルです。

## 大原則

- **tasklist.mdが正式なドキュメント**。TodoWrite等の内部タスク管理は補助であり、進捗は必ずtasklist.mdに反映する
- **全タスクが `[x]` になるまで作業を継続する**
- **完了マーク(`[x]`)は操作の実行後に付ける**。実行予定の段階で付けない(コミット・PR作成など、マークの後に実行しがちな操作は特に注意。マークと実態の乖離はフックでは検出できない)
- タスクの完了・スキップはリアルタイムにEditツールで記録する(まとめて更新しない)

### フックによる担保

このプロジェクトでは以下がフックで機械的に強制される(`.claude/hooks/`):

- **Stopフック**: 最新ステアリングディレクトリのtasklist.mdに `[ ]` が残っていると、セッション終了がブロックされる
- **PostToolUseフック**: 実装ファイルの編集が続いてtasklist.mdが更新されないと、リマインドが注入される

ブロックされた場合は、タスクを完了させるか、下記ルールA/Bを適用してからセッションを終了する。

### ルールA: タスクが大きすぎる場合

タスクを小さなサブタスクに分割してtasklist.mdに追加し、1つずつ完了させる。

### ルールB: 技術的理由によるスキップ

スキップが許されるのは技術的理由(実装方針の変更・アーキテクチャ変更・依存関係の変更により不要になった)のみ。「時間の都合」「難しい」等は不可。

```markdown
- [x] ~~タスク名~~（実装方針変更により不要: 具体的な技術的理由）
```

スキップ理由は振り返りセクションにも記録する。

## モード1: ステアリングファイル作成(計画)

1. 現在の日付で `.steering/[YYYYMMDD]-[タスク名]/` を作成する
2. 永続ドキュメント(`docs/` の product-requirements / functional-design / architecture / repository-structure / development-guidelines)を読み、方針を理解する
3. 以下のテンプレートを読み込み、プレースホルダーを具体化してファイルを作成する:
   - `.claude/skills/steering/templates/requirements.md` → `requirements.md`(関連Issue URLを必ず記載)
   - `.claude/skills/steering/templates/design.md` → `design.md`
   - `.claude/skills/steering/templates/tasklist.md` → `tasklist.md`
4. tasklist.mdはrequirements.md / design.mdに基づいて詳細化する:
   - 実装可能なタスクのみをリストアップする(「将来やるかも」「検討中」は含めない)
   - フェーズ分け・実装順序・サブタスクを明確にする

## モード2: 実装

1. tasklist.mdを読み、次の未完了タスク(`[ ]`)を特定する
2. 実装する(docs/development-guidelines.md の規約に従う)
3. 完了したら**即座に**Editツールで該当行を `[ ]` → `[x]` に更新する(サブタスクも個別に)
4. 1に戻る。フェーズ完了時はtasklist.md全体を読み直し、見落としがないか確認してユーザーに報告する

計画と実装が乖離した場合:
- 該当タスクに注釈を付ける(例: `- [x] タスク名（実装方法を変更: 理由）`)
- 必要なら新タスクを追加し、設計変更が大きい場合はdesign.mdも更新する

## モード3: 振り返り

全タスクが `[x]` になったことを確認してから、tasklist.mdの「実装後の振り返り」セクションをEditツールで更新する:

- 実装完了日
- 計画と実績の差分(計画と異なった点・追加タスク・ルールBのスキップ理由)
- 学んだこと(技術面・プロセス面)
- 次回への改善提案
- リリース判断(テンプレートの表に従って評価・提案)

振り返りは蒸留(platform-harnessへの環流候補の抽出)の入力になるため、他プロジェクトでも通用する知見(普遍性・再現性・抽象性)を意識して書く。

記録後、ユーザーに報告する:
「振り返りをtasklist.mdに記録しました。内容を確認してください。」
