## 実装アプローチ

`docs/repository-structure.md` を以下の構成に再編成する:

1. **先頭に Bootstrap フェーズ構造セクションを追加** — このリポジトリ自体の現在の構造
2. **既存の「完成形」セクションを `kiro-template/` 配下として位置づけ直す** — Kiro ユーザーへの展開物
3. **既存のディレクトリ詳細セクションのパスを更新** — `kiro-template/` プレフィックスを追加

## 新しい文書構造

```
## このリポジトリの構造（Bootstrap フェーズ）
  → このリポジトリ全体の実際の構造ツリー

## kiro-template/ の構造（Kiro テンプレートの成果物）
  → kiro-template/ 配下のツリー（既存「完成形」を流用）

## ディレクトリ詳細
  → 各ディレクトリの役割説明（パスを kiro-template/ 配下に更新）

## テンプレートとしての使い方
  → kiro-template/ の使い方に合わせて更新
```

## 技術的判断と根拠

| 判断 | 根拠 |
|------|------|
| `.devcontainer/` を「Claude Code 開発環境」として明記する | 設計指針2（devcontainer 非依存）はKiro テンプレート側の話。このリポジトリの開発環境としての `.devcontainer/` は矛盾しない |
| `.steering/YYYYMMDD-xxx/` を「作業履歴」として記載する | Claude Code 作業スペックはこのリポジトリの意思決定履歴として価値がある |
| 既存の「完成形」セクションタイトルを「kiro-template/ の構造」に変更する | `kiro-template/` が成果物フォルダであることを明確にする |

## 実装の順序

1. `docs/repository-structure.md` の先頭に Bootstrap フェーズ全体構造セクションを追加
2. 既存「プロジェクト構造（完成形）」セクションのタイトル・ツリーを更新
3. 「ディレクトリ詳細」の各セクションパスを `kiro-template/` 配下に更新
4. 「テンプレートとしての使い方」のカスタマイズ対象パスを更新
