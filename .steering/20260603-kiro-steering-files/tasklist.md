## 実装後の振り返り

### 実装完了日
2026-06-03

### 計画と実績の差分

**計画と異なった点**:
- 実装中に `kiro-template/` アウトプット専用フォルダを導入することを決定した。当初は `.kiro/` をリポジトリルートに直接作成する想定だったが、Claude Code 開発インフラ（`.claude/`・`CLAUDE.md`）との混在を避けるため変更した
- これに伴い、全 Kiro 成果物の配置パスが `kiro-template/` 配下に変更された

**新たに必要になったタスク**:
- `docs/repository-structure.md` の更新（`kiro-template/` フォルダを反映）→ 次スペックで対応

### 学んだこと

**技術的な学び**:
- Kiro の `inclusion: always` フロントマターキーは `inclusion`（`inclusionMode` ではない）。設計書では `inclusionMode` と記述していたが、Kiro の実際の仕様に合わせた
- `process.md` はプロジェクト固有のプレースホルダーを含めない（ハーネス共通のルールブック）。他3ファイルとは性格が異なる

**プロセス上の改善点**:
- 実装開始前に「アウトプットフォルダをどこに切るか」を決定してから実装に入ると、パス変更の手戻りが発生しない
- `kiro-template/` の導入という設計判断は、ブートストラップフェーズ固有の判断として memory に記録済み

### 次回への改善提案
- `docs/repository-structure.md` を `kiro-template/` 構造に合わせて更新する
- `kiro-template/.steering/_template/` の実装（次スペック）
- `kiro-template/.kiro/hooks/` の実装（次スペック）
