# 要求内容

## 概要

`docs/product-requirements.md` をリポジトリの現在の実態（マルチテンプレート構成）に合わせて更新する。

## 背景

PRD は `platform-harness-for-kiro` の実装フェーズで作成されたもので、プロダクト名・ビジョン・機能要件がすべて「Kiro 向けハーネステンプレートを作る」という想定で記述されている。その後 `codex-template/` が追加され、リポジトリの実態は「複数の AI ツール向けハーネステンプレートを管理・配布するプラットフォーム」に変化した。

関連 Issue: https://github.com/kanan4gh/platform-harness-engineering/issues/29

## ユースケースの軸

ハーネスエンジニアが PRD を読んだとき、「このリポジトリは何を目的とし、どの AI ツール向けテンプレートを管理しているか」が一目でわかる。

## 実装対象の機能

### 1. プロダクト概要の再定義

- プロダクト名を `platform-harness-engineering` として再定義する
- ビジョンを「特定ツール向け」から「複数の AI ツールに横断的なハーネス設計・配布プラットフォーム」に変更する
- プロダクトコンセプトにマルチテンプレート構成（Kiro / Codex CLI、今後も追加可能）を反映する

### 2. 機能要件の更新

- 既存の Kiro テンプレート要件を整理・残す
- Codex CLI テンプレート（`codex-template/`）の要件を追加する
- テンプレート共通インフラ（publish ワークフロー等）を要件として追加する

### 3. 将来的な機能の更新

- 「新ツール向けテンプレートの追加プロセス」を将来的な機能として追加する
- 既存の Post-MVP 項目を現在の実態に合わせて見直す

## 受け入れ条件

### プロダクト概要の再定義
- [ ] PRD のプロダクト名が `platform-harness-engineering`（またはそれに相当する名称）になっている
- [ ] ビジョンに Kiro / Codex CLI の両テンプレートを扱うことが明示されている
- [ ] 将来的に他ツール向けテンプレートを追加できる拡張性が示されている

### 機能要件の更新
- [ ] Kiro テンプレート要件が残っている
- [ ] Codex CLI テンプレート（`codex-template/`）が機能要件として記載されている
- [ ] テンプレート配布自動化（publish ワークフロー）が要件として記載されている

### 将来的な機能の更新
- [ ] 「新ツール向けテンプレートの追加プロセス」が Post-MVP に記載されている

## スコープ外

- `docs/` 以外のファイル変更（README 等は別タスク）
- 機能設計書・アーキテクチャ等の他ドキュメントの更新
- 新機能の実装

- `codex-template/` - Codex テンプレートの実態
